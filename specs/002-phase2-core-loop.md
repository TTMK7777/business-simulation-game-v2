# 002 — Phase 2 遊びの骨格

策定日: 2026-07-20 ｜ 対応ペルソナ: `docs/PERSONAS.md` ②田中（緊張感）③中村（短尺）中心
背景: 1クリック=1週・月次決算は第4週のみのため、4週中3週が無イベントの空振りでペルソナ共通核「打ち手→数字の因果の体感」が希釈されている。`employeeResignChance`（gameConfig.ts 定義済み）は未使用、`burnout_prone`/`inconsistent` 特性も効果がなく、打ち手を誤るリスク＝緊張感が存在しない。現状はクリア条件のないサンドボックスで起承転結がない。

## 設計判断（2026-07-20 ユーザー確定）

| 論点 | 決定 | 補足 |
|---|---|---|
| テンポモデル | **B: 週次ミニイベント充填** | 週構造は維持し、空白週にコンテンツを充填する |
| 初回シナリオ | **起業1年目** | 12ヶ月生存。既存の資金繰り・破産機構で成立し追加機構が最小 |
| CEO前面化 | **決裁カードを管理モードに注入** | 決裁カードが週次イベントの主力になり、B案の懸念（コンテンツ量産）を既存 CEO モード資産の転用で緩和する |

## スコープ

### A. 週次ミニイベント基盤
- 決算週以外の各週で、イベントプールから抽選して1件提示する（週あたり最大1件 — 水増しノイズ化の防止）
- イベント種別: ①決裁カード（C）②人事イベント（B の予兆・面談）③市場・ニュース選択イベント（既存ニュースの選択肢付き拡張）
- **設計制約（共通核）**: 全イベントは選択結果の「数字への影響」を明示する（impact 表示は Phase 1 の a2ui 資産を流用）

### B. 退職・モチベーションの実効化
- motivation の変動源を整理（過重労働・放置・研修・決裁結果）し、`employeeResignChance` をモチベーション連動で接続
- `burnout_prone` / `inconsistent` 特性を実効化
- 予兆の可視化: アドバイザーカード（危険水域警告）+ キャラ stressed 状態（いずれも Phase 1 資産の転用）
- 退職は数字で見える損失にする（再採用コスト・開発力低下 → 売上ドライバーに反映）

### C. CEO決裁カードの管理モード注入
- DocumentManager 資産を管理モードの週次イベント（A のプール）として注入
- 決裁結果の理論タグ表示 + 図鑑解禁（v2.3.0 資産）を管理モードでも発火させる
- 訪問者システム等の CEO 専用機能は CEO モード据え置き（モード自体は統合しない）

### D. シナリオ「起業1年目」
- 新シナリオモード: 初期資金を絞った12ヶ月生存。資金ショート＝ゲームオーバー、12ヶ月生存＝クリア
- クリア/ゲームオーバー画面 + 事後講評の最小版（主要指標サマリ + 体験した理論の一覧。バックログ「経営を理論で振り返る」の v1）
- 既存の破産判定・難易度機構を流用

### E. 固定費（トレードオフの最小実装）
- オフィス維持費を月次固定費として P/L に計上（officeLevel 連動）
- D の資金繰り緊張感の燃料であり、Phase 1 の財務3表可視化と直結する

## 実装順序（依存関係）

1. **Wave 1: B + E**（managers 層のみ、UI 変更最小 — 緊張感の核）
2. **Wave 2: A + C**（週次イベント基盤 + 決裁注入。B の人事イベントをプールに接続）
3. **Wave 3: D**（シナリオ + クリア/講評画面。A-C が揃ってから12ヶ月の体験曲線を調整）

## 非スコープ

- 市場セグメント + PPM 実UI（Phase 3、テンポと統合設計必須のため）
- シナリオ「部門P/L再建」「上場」（起業1年目の実装パターン確立後に横展開）
- モバイル対応・収益化（Phase 3）

## 検証

- Wave ごとに Vitest（新規ロジックはテスト先書き）+ tsc 0 を維持
- UI 変更は実ブラウザ E2E 必須: `scripts/e2e-wave1.mjs` を Phase 2 チェック（週次イベント発火 / 退職発生 / シナリオクリア）へ拡張
- バランス検証: 起業1年目を実プレイ相当の自動進行で N 回走らせ、クリア率が極端（<10% / >90%）でないことを確認
