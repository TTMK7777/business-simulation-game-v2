# コントリビューションガイド

ビジネスエンパイア 2.0（IT業界経営シミュレーション）へのコントリビュートに興味を持っていただき、ありがとうございます。本プロジェクトは MIT ライセンスで公開されているオープンソースソフトウェアです。バグ報告・機能提案・コード改善など、どんな形の貢献も歓迎します。

参加にあたっては [行動規範 (CODE_OF_CONDUCT.md)](./CODE_OF_CONDUCT.md) の遵守をお願いします。

## Issue の報告

バグ報告や機能要望は [GitHub Issues](https://github.com/TTMK7777/business-simulation-game-v2/issues) からお願いします。

### バグ報告に含めてほしい情報

- 再現手順（できるだけ具体的に、番号付きで）
- 期待した挙動と実際の挙動
- 実行環境（OS、ブラウザとそのバージョン、Node.js のバージョン）
- 可能であればスクリーンショットやコンソールのエラーログ

### 機能要望

- 解決したい課題・ユースケース
- 想定する挙動や UI のイメージ
- 既存機能との関係（代替手段の有無など）

## 開発環境のセットアップ

Node.js（推奨 LTS 以降）が必要です。

```bash
npm install        # 依存関係のインストール
npm run dev        # 開発サーバー起動
npm run dev:sp     # LAN公開モード（スマホ実機確認用）
npm test           # テスト実行（Vitest, watchモード）
npm test -- --run  # テストを1回だけ実行（CI向け）
npm run build      # 本番ビルド
npm run preview    # ビルド成果物のプレビュー
```

## プルリクエストの流れ

1. `main` ブランチから作業ブランチを分岐します（例: `feat/...`、`fix/...`、`docs/...`）。
2. `main` への直接 push は禁止です。必ずプルリクエスト経由で変更を取り込みます。
3. 1つの PR は1つの目的に絞ってください（無関係な変更を混ぜない）。
4. コミット前に `npm test -- --run` を実行し、全テストが通ることを確認してください。
5. PR の説明には目的・変更内容・確認方法を記載してください。

## コーディング規約

- 既存のコードスタイル・ファイル構成を踏襲してください。
- TypeScript は `strict: true` 前提です。`noUnusedLocals` / `noUnusedParameters` も有効なため、未使用の変数・引数を残さないでください。
- UI は Lit 3 + テンプレートリテラル（段階移行中）に従います。
- ユーザー入力は既存方針どおりサニタイズ／エスケープしてください（`escapeHtml` 等）。
- 変更にはできる限り Vitest のテストを追加・更新してください。
